## 1. What's actually built (more than we gave it credit for)

**The live 5-minute cycle** is a genuinely deep pipeline, all verified wired:
regime classifier (SPY/VIX/credit spreads) → dynamic S&P momentum watchlist (sector-capped)
→ technical indicators → ten enrichers (news sentiment, options flow, EDGAR insider clusters,
macro/FRED, VWAP tick agent, economic calendar, earnings, FedWatch, trade-history RAG,
performance feedback) → Claude decision → **validation agent (active second-LLM devil's
advocate on every buy; hard-blocks confidence <0.65 or size >12%)** → guarded execution
→ reconciler every 30s (fills, brackets, SMA50 exit monitor, post-exit "left on the table"
tracking, signal calibration) → as of today: scorecard, daily digest, Slack alerts.

**Guards confirmed in the execution path** (more complete than ROADMAP believed):
confidence ≥0.72 · pending-order check (fail-closed) · max 8 positions · correlation block
at 0.75 · catalyst risk tiers 0–3 · **PDT guard (exists! blocks ≥3 day-trades, live-mode only
— ROADMAP listed it as unbuilt)** · **daily-loss circuit breaker (5%, exists — halts new
entries)** · 80% deployment cap · 25% single-position cap · ATR trailing stops (12–25%).

**The self-improvement loop exists end-to-end** (calibration → prompt injection → "you've
been overconfident" warnings) but is **data-starved**: it only learns on position closes and
there have been just 31 ever.

**Dashboard:** 10 pages, all functional, cleanly decoupled through the API. No direct DB reads.

## 2. Corrections to prior beliefs

- **PDT guard**: built (trader.py ~L268). ROADMAP P0 item already done.
- **Daily-loss circuit breaker**: built (5% halt). VISION-RETHINK assumed missing.

## 3. The gaps, prioritized

**G1 — Position-cap contradiction (highest risk).** Three different caps coexist:
trader hardcodes 25%/position; manual-order paths hardcode 8%; DB config says 8% and is
ignored. The autonomous trader can take 3× the risk a manual order can. Pick one number,
one source of truth.

**G2 — Zero tests, zero CI.** 19k lines intended to manage real money; no test file exists,
no GitHub Actions. Any refactor (including the casino-model transplant) is unprotected.

**G3 — Execution leak.** IOC limit entries that place-then-cancel get no retry (~50% of
historical entries died this way). The market-order fallback only covers placement
*exceptions*, not cancels. Fill-rate SLO needed (VISION-RETHINK L3).

**G4 — Backtester is stale and orphaned.** Complete module, but tests pre-enricher strategy
rules and never writes backtest_runs. The live strategy has effectively never been validated
against history. This blocks Phase B (edge library needs a trustworthy backtest harness).

**G5 — Feedback loop starvation.** signal_calibration / trade_analysis / post-exit tracking
all fire only on closes (31 ever). The "self-improving" promise is real code waiting for data
volume the current 2-position posture can't generate.

**G6 — Config sprawl.** Four sources (.env, DB config, toml, legacy config.py) with real
contradictions: cap conflict (G1), llm cost computation can mismatch actual provider,
catalyst-risk thresholds hardcoded and untunable.

**G7 — Dead weight.** ~8 legacy modules (v1 engines: ai_engine, autonomous_trader,
trading_engine, auth, data_manager, options_scraper, wi_config + stale backtester) ≈3k lines
confusing every future reader/agent. Archive them.

**G8 — Built but switched off.** 13D activist / 8-K event EDGAR parsing is complete and
explicitly disabled in the enricher; FedWatch runs on hardcoded fallback; news sentiment is
a word-list, not a model. Free signal value sitting dormant.

## 5. Recommended sequence (boring, evidence-first)

1. **G1 cap unification** — one config key, one enforcement point, hard-coded guard. (hours)
2. **Smoke-test suite + CI** — guards, sizing, calendar, scorecard; GitHub Actions on push. (a day)
3. **G3 fill-rate fix** — marketable-limit retry ladder + fill-rate in daily digest. (a day)
4. **G4 backtester refresh** — current rules, enricher-aware; prerequisite for Phase B edges. (days)
5. **Archive dead modules (G7), re-enable 13D/8-K (G8)** as the first new "edges" to measure.
6. Then Phase B per VISION-RETHINK: edge library A/B against the current brain.

*The system is a real platform with real rails — what it lacks is proof: tests for the code,
backtests for the strategy, and fills for the signals.*
